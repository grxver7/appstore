Tinder-like matching platform for financial and insurance brokers

Frontend :5173 Backend :8020 FastAPI React 18 TypeScript Zustand SQLite
Quick Start
⚡ One-command start: After the first-time setup below, use start.sh to launch both servers at once.
→

cd ~/apps/maklermatch bash start.sh
1
First-time: set up the backend
Create a virtual environment, install dependencies, and configure environment variables
# From the project root cd ~/apps/maklermatch/backend python3.12 -m venv .venv source .venv/bin/activate pip install -r requirements.txt cp .env.example .env
2
Initialize the database & seed demo data

alembic upgrade head python seed.py
3

4
Start the frontend
Open a new terminal — Vite dev server on port 5173
cd ~/apps/maklermatch/frontend npm install cp .env.example .env npm run dev

Broker Profiles Upload photos, pitch video (file or YouTube/Vimeo URL), certifications and specializations

Smart Feed with Ranking Score = 0.6 × tag overlap + 0.25 × rating + 0.15 × recency. Hard filters for region and language
💬
WebSocket Chat Real-time messaging on match. JWT auth via query param for WebSocket handshake
⚖️
DSGVO Compliant DELETE /users/me cascades to all profiles, swipes, matches, messages, reviews and uploads
🌟
Reviews & Ratings Clients can rate and review matched brokers. Ratings feed back into the ranking algorithm

Assurance Levels
🟢
L1 Casual Simple majority or approval vote. No email verification required. Quick polls for informal groups.
🔵
L2 Verified Email verification, secret ballot (optional), quorum + threshold, eligibility snapshot, hash-chain ledger, ballot receipts.
🟣
L3 High-Assurance All L2 features + weighted and quadratic voting, liquid delegation (depth 10), observer role, 2FA (TOTP), audit export.
Tally Methods
✅
Simple MajorityOne option, most votes wins. Tie → no winner declared.
☑️
ApprovalCheck any number of options — highest approval count wins.
🔢
Ranked Choice (IRV)Instant-runoff elimination rounds. Deterministic tie-break.
⭐
ScoreRate each option 0–5. Highest average wins.
⚖️
WeightedVotes scaled by voting_weight (e.g. share count).
🔬
QuadraticCredit budget — cost = votes². Spend across multiple options.

NPL Network

Self-hosted personal contact & relationship management API

API :8023 FastAPI Async SQLAlchemy SQLite / PostgreSQL D3.js Graph
Quick Start
ℹ️ API only — no dedicated frontend. Use the Swagger UI at localhost:8023/docs or connect any HTTP client.
⚡ One-command start: After the first-time setup below, use start.sh to start the backend.
→
Subsequent runs — one-command start
Starts the backend on :8023 in one go
cd ~/apps/npl bash start.sh
1
First-time: configure and install
Create .env with a generated API key
cd ~/apps/npl cp .env.example .env # Generate a secure API key and paste it into .env python3 -c "import secrets; print(secrets.token_urlsafe(32))" python3 -m venv venv source venv/bin/activate pip install -r requirements.txt
2
Initialize the database
Creates all tables and seeds one example contact
mkdir -p app/data app/logs python3 database_init.py
3
Start the server on port 8023
Async FastAPI — all endpoints available immediately
uvicorn app.main:app --host 0.0.0.0 --port 8023
4
Make your first API call
All protected routes require the X-API-Key header
# Check health (no auth) curl http://localhost:8023/api/health # List contacts (requires API key) curl -H "X-API-Key: YOUR_KEY" \ http://localhost:8023/api/entities
API Routes
👥
Entities (Contacts) Full CRUD — name, profession, city, rating 1–5, interests, notes, birth date, contact methods (email, phone, linkedin, instagram, twitter, website)
🔗
Relationships Directed graph edges — type (friend, colleague, mentor, …), strength 1–10, last interaction date, notes
🔍
Full-Text Search GET /api/search?q=term — searches names, city, profession, notes, interests. Filter by city or profession.
📊
Network Graph GET /api/network-graph → D3.js-ready {nodes, edges} payload for visualization
📋
Activity Log & Stats Recent create/update/delete events and aggregate statistics across all contacts
Raspberry Pi Deployment
1
Create systemd service
Auto-start on boot at /etc/systemd/system/npl.service
[Unit] Description=NPL Backend After=network.target [Service] User=pi WorkingDirectory=/home/pi/npl ExecStart=/home/pi/npl/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8023 Restart=on-failure EnvironmentFile=/home/pi/npl/.env [Install] WantedBy=multi-user.target
2
Enable and start
Follow logs with journalctl
sudo systemctl daemon-reload sudo systemctl enable npl sudo systemctl start npl sudo journalctl -u npl -f

Market Analyzer

AI investment planner — runs 100% locally with Ollama. Free forever.

Frontend :5176 Ollama LLM React Recharts Tailwind CSS No Backend
Quick Start
🤖 Requires Ollama. Install it first — the LLM runs entirely on your machine. No API key needed for inference.
→
Subsequent runs — one-command start
Starts the Vite dev server on :5176 (make sure Ollama is running separately)
cd ~/apps/investments bash start.sh
1
First-time: install Ollama
One-liner for Linux/WSL2. Download installer for Windows/Mac from ollama.com.
# Linux / WSL2 curl -fsSL https://ollama.com/install.sh | sh
2
Pull a model
Choose based on your available RAM — qwen2.5:7b recommended for most systems
ollama pull qwen2.5:7b # 4.4 GB — needs 8 GB RAM (recommended) ollama pull qwen2.5:14b # 8.4 GB — needs 16 GB RAM ollama pull qwen2.5:32b # 19 GB — needs 32 GB RAM
3
Start Ollama — keep this terminal open
Must be running before you use the app. Use the remote-access variant if deployed behind a domain.
# Local use (Linux / WSL2) OLLAMA_ORIGINS="*" ollama serve # Remote access (deployed to a domain) OLLAMA_ORIGINS="https://yourdomain.com" OLLAMA_HOST="0.0.0.0" ollama serve
4
Add optional API keys for live market data
The app works without these — live benchmark prices just won't be injected into the AI prompt
cd ~/apps/investments cp .env.example .env.local # or create manually # Edit .env.local and fill in what you have: VITE_ALPHA_VANTAGE_KEY=YOUR_KEY # alphavantage.co (free tier) VITE_COINGECKO_KEY=YOUR_KEY # coingecko.com (free tier) VITE_FRED_KEY=YOUR_KEY # fred.stlouisfed.org (free)
5
Start the app on port 5176
Make sure Ollama is running before clicking "Create Investment Plan"
cd ~/apps/investments npm install npm run dev
Data Sources
🤖
Ollama (Local LLM) Generates the full portfolio plan. Default model: qwen2.5:7b. Override via VITE_OLLAMA_MODEL in .env.local.
🪙
CoinGecko Live crypto benchmark prices — current price, 24h/7d change, market cap, volume. Free tier available.
📉
Alpha Vantage Live stock/ETF benchmark prices, P/E ratio, 52-week range, sector data. Falls back to AI-only mode if rate limit hit.
🏦
FRED (Federal Reserve) US macro data: CPI inflation, Fed Funds rate, unemployment. Enriches every plan with real economic context.

HumMatch

Hum or sing a melody — identify the song. Melody-contour matching, not audio fingerprinting.

Frontend :5177 API :8030 ACRCloud React + Vite FastAPI SQLite Framer Motion
Quick Start
🔑 Requires ACRCloud credentials. Create a Humming Recognition project (not Audio Fingerprint) at console.acrcloud.com and paste the Host / Key / Secret into backend/.env.
→
Subsequent runs — one command
Starts both backend (:8030) and frontend (:5177)
cd ~/apps/hummatch ./start.sh
1
Set up the Python venv
Uses Python 3.12 — not the system default
cd ~/apps/hummatch/backend python3.12 -m venv .venv .venv/bin/pip install -r requirements.txt
2
Add ACRCloud credentials to .env
Copy the example file and fill in your Humming Recognition project details
cd ~/apps/hummatch cp .env.example backend/.env # Edit backend/.env: ACRCLOUD_HOST=identify-eu-west-1.acrcloud.com ACRCLOUD_ACCESS_KEY=your_key ACRCLOUD_ACCESS_SECRET=your_secret
3
Install frontend dependencies
cd ~/apps/hummatch/frontend npm install
How It Works
🎤
Record 5–12 seconds Tap the button and hum or sing any part of the melody. The browser validates duration and volume before uploading — no wasted API calls on silence.
🌊
Real-time waveform 32 bars driven by the Web Audio API AnalyserNode during recording — shows the mic is actually picking up sound.
🎯
Ranked results with confidence Returns up to 5 candidates sorted by match score. Below 60% confidence, shows a "Not sure" state instead of pretending certainty.
🔌
Swappable engine interface MatchEngine ABC in match_engine/base.py — swap ACRCloud for another provider without touching the rest of the app. Audd.io stub already included.
